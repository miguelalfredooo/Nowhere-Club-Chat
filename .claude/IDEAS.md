# Ideas & Future Features

## AI Feedback Analysis & Inter-User Knowledge Sharing
**Status:** Concept (not in current scope)

When users provide feedback on media/objects in the archive, AI analyzes whether that feedback is **additive to core concepts** — does it refine, expand, or contradict the fundamental understanding of what that object represents?

### Two-Way Knowledge Flow
- **User → Archive**: User feedback enriches the concept model
- **Archive → User**: When User B is chatting about an object, AI brings up relevant insights from User A's previous feedback on the same or related objects
- Creates a **collective knowledge layer** without needing explicit connections

### Implementation Notes
- Feedback must be validated as concept-relevant (not spam/noise)
- Privacy considerations: surface insights without exposing user identity unless intentional
- Could apply to cross-object relationships too (e.g., "other users connected X and Y in interesting ways")
- Vision AI + conversation context needed to determine relevance

### Why It Matters
- Transforms archive from personal collection → collaborative thinking tool
- Surfaces unexpected connections between users' perspectives
- Deepens concept understanding through multiple lenses
